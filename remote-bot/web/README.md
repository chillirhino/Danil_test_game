# 🦫 Web Console — терминал Claude в браузере телефона

Страница-терминал: пишешь задачу → видишь весь поток работы (текст, команды, код правок, результаты). Добавляется на домашний экран как приложение (PWA).

## Файлы
- `server.js` — локальный веб-сервер на ПК (без зависимостей). Стримит вывод Claude через SSE.
- `index.html` — сам терминал (тёмная тема, вход по паролю).
- `start-web.bat` — запуск с авто-перезапуском.

Настройки берутся из `../config.json`: `webPort` (8765), `webPassword`, а также `projectDir`, `skipPermissions`, `claudeBin`, `timeoutMinutes`, `showToolResults`.

## Запуск
1. В `../config.json` поменяй **`webPassword`** на свой (обязательно!).
2. Двойной клик по `start-web.bat`. При первом запуске Windows может спросить доступ node к сети — **Разрешить** (частные сети).
3. На телефоне **в той же Wi-Fi** открой в браузере `http://<IP-ПК>:8765/` (IP печатается при запуске), введи пароль.
4. Меню браузера → «Добавить на главный экран» — будет как приложение.

## Доступ откуда угодно (не только дома)
Сервер слушает всю локальную сеть. Чтобы открывать с мобильного интернета:
- **Tailscale** (рекомендую): поставь на ПК и телефон (есть Android-приложение), войди одним аккаунтом → открывай `http://<tailscale-IP-ПК>:8765/`. Безопасно, наружу не торчит.
- Или публичный туннель (cloudflared/ngrok) на порт 8765 — тогда пароль критичен.

## Безопасность
- Доступ к API (стрим и отправка) — только с верным `webPassword`. **Смени дефолтный пароль.**
- Это удалённое управление ПК (Claude запускает команды и меняет файлы). Не открывай наружу без пароля/туннеля.
- Веб-консоль и Telegram-бот независимы — пользуйся любым. Не гоняй одну и ту же задачу в обоих одновременно.
